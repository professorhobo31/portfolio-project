Esta página constituye el primer acercamiento, de mi parte, al trabajo con Bootstrap. El proyecto usa una estructura de archivos .scss, y un archivo _custom.scss que edita ciertas variables base de bootstrap (la selección de theme colors por ejemplo) sin editar destructivamente los archivos base de Bootstrap. Destaco además:

Flexbox

Esta sección ha sido estilizada usando propiedades flex de manera tal que en pantallas grandes el texto y las imágenes vayan intercambiando las mitades horizontales de la pantalla, pero que al disminuir al tamaño de las pantallas que usan los celulares, el orden no cambie y se vuelva un patrón lineal texto-imagen.

Uso de Sass

La estructura de archivos del proyecto se dividió en dos carpetas principales de archivos .scss. Una contiene una lista de componentes: la idea al inicio era agrupar aquí cualquier animación, mixin o estilo que fuera a reusarse. Otra carpeta contiene una lista de seccionGulp y es: esto permite desglosar y trabajar en los estilos usando archivos menos sobrecargados y ayuda a organizar.

Gulp y Browsersync

Los archivos .scss finalmente se compilan y minifican a través de Gulp. Leyendo la documentación de Gulp se creó el archivo gulpfile.js, creando un comando que active un servidor falso de desarrollo mediante el paquete Browsersync. Así, ante cada edición de los archivos .scss o index.html, se regeneran automáticamente los archivos de output .css y la página se recarga en vivo, acelerando el proceso de desarrollo.
