Esta página constituye un primer acercamiento, de mi parte, al trabajo con Bootstrap. El proyecto usa una estructura de archivos .scss para editar estilos, y en particular un archivo _custom.scss que edita ciertas variables base de bootstrap (la selección de theme colors por ejemplo) sin editar destructivamente los archivos base de Bootstrap. Destaco un breve resumen de lo practicado en el proyecto:

Flexbox

Esta sección ha sido estilizada usando propiedades flex de manera tal que en pantallas grandes el texto y las imágenes vayan intercambiando las mitades horizontales de la pantalla, pero que al disminuir al tamaño de las pantallas que usan los celulares, el orden no cambie y se vuelva un patrón constante texto-imagen.

Uso de Sass

Para dar estructura al proyecto se dividió en dos carpetas principales de archivos .scss. En src/scss/components hay una lista de componentes: se agrupa aquí cualquier animación, mixin o estilo que se intenda reusar en el proyecto. La carpeta src/scss/sections contiene una lista de secciones: esto permite desglosar y trabajar en los estilos usando archivos con menos líneas de código y ayuda a organizar.

Gulp y Browsersync

Los archivos .scss finalmente se compilan y minifican a través de Gulp. Leyendo la documentación de Gulp se creó el archivo gulpfile.js, creando un comando que active un servidor falso de desarrollo mediante el paquete Browsersync. Así, ante cada edición de los archivos .scss o index.html, se regeneran automáticamente los archivos de output .css y la página se recarga en vivo, acelerando el proceso de desarrollo.
